## Quick Start

> **Standalone component**: yes

```typescript
import { RouterLinkWrapperComponent } from '@extrawest/extra-clarity/router-link-wrapper';
```
```html
<ec-router-link
  [enabled]="driverId && tenantId"
  [fwdRouterLink]="['/drivers', driverId, tenantId]"
>
  {{ driverId }}
<ec/>
```

## Description

This component was designed to prevent showing and clicking on router links broken due to missed parts of their urls,
leading to non-existent pages.

It uses `<a [routerLink]>` internally when the `[enabled]` input is `true`, and passes its own content into that `<a>`
element. Otherwise, it shows the content as it is without any wrappers.

Although broken links could be disabled by passing `null` or `undefined` to the original `[routerLink]`:
```html
<a [routerLink]="driverId && tenantId ? ['/drivers', driverId, tenantId] : null">
```
it may look cumbersome and does not prevent the `<a [routerLink]>` element from being shown as a link on hovering and
clicking, due to the applied global styles (see the last line in the example above).
If this behavior seems like intended for you, then you should go with the plain `<a [routerLink]>`.
